Page Change Monitor Agent

Week 4 M13 · Agentic AI Solo Agent
Claude.ai subscription required

Requires Claude.ai Pro or Team with Cowork access. Open Claude, start a Cowork session, and paste the agent prompt below. No API key or local Python setup needed.

What this does

This agent monitors a list of web pages you care about — competitor pricing, status pages, changelogs, job boards — and tells you exactly what changed between checks. It:

  • Visits each URL in your watch list
  • Compares the current content to the last-seen version
  • Produces a clear diff summary for each changed page
  • Highlights whether changes are significant (pricing, breaking changes, outages)
  • Saves an alert log so you have a history of all changes

Useful for monitoring competitor pricing pages, SaaS changelogs, status pages, or any page where changes matter.

Subscription required

Claude.ai Pro or Team subscription with Cowork access is required. Cowork lets Claude browse the live web and compare page contents across multiple URLs.

Claude.ai Pro Claude.ai Team Cowork feature
Step-by-step in Claude Cowork
Open claude.ai and sign in to your Pro or Team account.
Click "Cowork" in the left sidebar to start a new Cowork session.
Decide which URLs you want to monitor. Good candidates: pricing pages, status pages, documentation changelogs, job listings, or competitor landing pages.
Paste this prompt into Cowork, updating the URL list with your own pages:
You are a page change monitor agent. I'll give you a list of URLs to check. Complete all steps WITHOUT asking me questions. URLs to monitor: - https://stripe.com/pricing - https://www.anthropic.com/news - https://status.github.com For EACH URL, complete these steps: 1. Browse to the URL and read its current content 2. Note the key facts visible on the page (prices, status messages, recent posts, etc.) 3. Identify anything that looks like it could have recently changed (new entries, price changes, status alerts) 4. Write a change summary for that page: what the page currently says, and flag any notable items After checking all URLs: 5. Write a consolidated "Change Report" with one section per URL: - URL - Current status: [what the page shows] - Notable items: [anything worth flagging] - Change risk: LOW / MEDIUM / HIGH (HIGH = pricing changed, breaking changes, outage) 6. Save the report as page-monitor-report.md Work through all URLs methodically. Flag anything significant.
Replace the example URLs with the pages you actually want to monitor.
For recurring monitoring, run the same prompt in a new Cowork session each day or week. Compare Claude's reports to spot what changed between runs.
